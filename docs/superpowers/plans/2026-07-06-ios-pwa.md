# Polimento iOS do PWA — plano de implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PWA instalado no iPhone com ícone certo, tela cheia e splash escura.

**Architecture:** PNGs gerados uma vez (script descartável puppeteer-core + Edge, fora do repo) e commitados; `index.html` ganha links/metas iOS; manifest ganha os PNGs. Sem dependência nova.

**Tech Stack:** puppeteer-core + Edge headless (geração), HTML/manifest.

## Global Constraints

- Desenho do ícone intocado (só rasterizar; versão full-bleed = mesmo SVG com `rx=0`).
- Splash `#070c18` com ícone 256px centrado; tamanhos: 1170×2532, 1179×2556, 1290×2796, 1125×2436, 828×1792 em `splash/`.
- Metas: `apple-mobile-web-app-capable=yes`, `apple-mobile-web-app-status-bar-style=black-translucent`, `apple-mobile-web-app-title=Minhas Obras`.
- `sw.js` bump `obras-v7` → `obras-v8`.

### Task 1: Gerar PNGs

- [ ] Script `%TEMP%\smoke-obras\gen-icons.mjs`: carrega `icon.svg` do repo, injeta em página HTML, screenshot em 180/192/512 (full-bleed: substituir `rx="112"` por `rx="0"`) → `apple-touch-icon.png`, `icon-192.png`, `icon-512.png` na raiz do repo; e páginas `#070c18` com o SVG original 256px centrado nos 5 tamanhos → `splash/splash-<W>x<H>.png`.
- [ ] Conferir dimensões dos PNGs gerados (ler bytes com `System.Drawing` ou abrir um no Read).
